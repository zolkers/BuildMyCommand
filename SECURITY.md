# Security Policy

## Supported Versions

Security fixes target the latest released version and the current development branch.

| Version | Supported |
| --- | --- |
| Latest release | Yes |
| `ddev` | Best effort |
| Older releases | No |

## Reporting a Vulnerability

Please report security issues privately when possible. If no private contact is available, open a GitHub security advisory or a minimal issue that avoids exploit details.

Include:

- Affected version or commit.
- A short description of the impact.
- Reproduction steps or proof of concept when safe to share.
- Any known workaround.

Maintainers will review reports as quickly as possible and coordinate a fix before public disclosure when appropriate.
